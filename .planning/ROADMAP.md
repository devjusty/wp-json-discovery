# Roadmap: WP JSON Discovery

## Phase 1: Scan Core
**Goal:** Make it easy to start a scan and immediately confirm WordPress evidence.
**Mode:** mvp
**Success Criteria**:
1. A user can enter a domain and trigger a scan from the main flow.
2. The result shows whether the site appears to be WordPress.
3. The result exposes wp-json evidence in a readable way.

### Requirements
- DISC-01
- DISC-02

## Phase 2: Signal Analysis
**Goal:** Turn scan output into a useful investigative surface for research and audits.
**Mode:** mvp
**Success Criteria**:
1. Plugin and theme clues are visible without hunting through raw output.
2. SEO and metadata indicators are grouped and easy to scan.
3. HTML/source clues are presented alongside the other evidence.

### Requirements
- SIGN-01
- SIGN-02
- SIGN-03

## Phase 3: Review and Ops
**Goal:** Support repeat investigations and feed operational signals back into the product.
**Mode:** mvp
**Success Criteria**:
1. A user can revisit prior scans from history.
2. Admin review surfaces unsupported namespace and logging signals.
3. The project has a clear feedback loop for expanding coverage later.

### Requirements
- REVW-01
- OPS-01
